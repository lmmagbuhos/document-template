## Essential Sections

### Header Elements (Required)

| Element | Required | Description |
|---|---|---|
| **TO** | Yes | Primary recipient(s) name and title |
| **FROM** | Yes | Sender's name and title, often with handwritten initials |
| **DATE** | Yes | The date of distribution or when the policy becomes effective |
| **SUBJECT** | Yes | A clear, concise statement of the topic indicating whether the memo is for action or information |
| **CC** | No | Names of others receiving copies |
| **MEMO/MEMORANDUM Title** | No | Optional centered title at the top |

### Body Sections

| Section | Required | Description |
|---|---|---|
| **Opening/Introduction** | Yes | Main point stated immediately |
| **Context/Background** | Yes | Historical or technical information reader needs |
| **Task/Purpose Statement** | Yes | What the document is about and why |
| **Discussion/Analysis** | Yes | Main content with evidence and data |
| **Summary/Executive Summary** | No | Road map for longer documents (1+ pages) |
| **Recommendations** | No | Specific, feasible, cost-effective actions |
| **Call to Action** | No | What reader must do, by when |
| **Closing/Conclusion** | Yes | Return to big picture, urgency, consequences |
| **Attachments/Enclosures** | No | Supporting charts, tables, data |

## BLUF Format (Bottom Line Up Front)

### Definition
A communication standard where the most important information is placed at the beginning — used originally in military and intelligence communities.

### Structure
- **Opening:** Main point, conclusion, or required action stated in first sentence
- **Follow-up:** Supporting details, context, and reasoning follow
- **Paragraph level:** First sentence of each paragraph summarizes all information in that paragraph

### Key Principles
- Prioritize reader's time
- Reduce cognitive effort
- Answer the "so what" immediately
- State purpose and required action clearly in opening
- Arrange information in descending order of importance
- Use deductive presentation style

### How to Write
1. Identify the "what" and the "so what" (key judgment and significance)
2. Answer the header's question directly in first sentence
3. State the main point before providing supporting evidence
4. Include who, what, where, when, and why in opening
5. Review conclusion for phrases like "In short" or "The key takeaway" and move to top
6. Delete throat-clearing, evidence-building setups, and meandering introductions

### Best Practices
- Be economical with words
- Use active voice
- Maintain clarity and precision
- Tailor to audience needs
- Provide enough context for reader to act without follow-up
- Ensure umbrella concept covers all information in paragraph

### When NOT to Use
- Delivering sensitive or bad news
- Cultures preferring indirect communication
- Complex topics requiring technical background first
- Building narrative tension in stories

## Formatting Guidelines

### Fonts
| Type | Options |
|---|---|
| **Recommended Serif** | Times New Roman, Georgia, Century Schoolbook, Baskerville |
| **Recommended Sans-Serif** | Arial, Calibri, Verdana, Helvetica, Century Gothic |
| **Standard Size** | 12 point minimum |
| **Enhanced Readability** | 14 point |
| **Headings** | Larger than body or bold weight |

### Margins
- **Standard:** 1 inch on all sides
- **Alternative:** 1 to 1.5 inches
- Consistency across entire document template

### Spacing
| Element | Specification |
|---|---|
| **Body text** | Single-spaced |
| **Between paragraphs** | Double-space (skip one line) |
| **Between heading and body** | Triple-space |
| **Line spacing alternative** | 1.5 or double for legal documents |
| **Paragraph style** | Block style with no first-line indentation |
| **Alignment** | Left justified (flush left) |
| **Length** | One page preferred, maximum two pages |
| **Title placement** | Centered (MEMO or MEMORANDUM, optionally bolded) |

### Visual Elements
- Headings and subheadings
- Bullet points and numbered lists
- Tables, charts, graphs
- White space (generous)
- Bold text
- Consistent capitalization

## Professional Standards Checklist

### Header Requirements
- [ ] Address reader by correct name and job title
- [ ] Use complete and current date
- [ ] Subject line is specific and clear enough for filing
- [ ] Handwritten initials or signature next to FROM line (for printed memos)

### Content Standards
- [ ] Problem is clearly articulated
- [ ] Purpose stated in opening paragraph
- [ ] Main point presented before supporting details
- [ ] Include only information reader needs
- [ ] Begin with most important information
- [ ] Use evidence-based recommendations
- [ ] Recommendations are feasible, cost-effective, and specific
- [ ] Address counterarguments and caveats
- [ ] Identify unintended consequences and impacted groups

### Structure & Organization
- [ ] Each paragraph contains only one main point
- [ ] First sentence of each paragraph is the topic sentence
- [ ] Information arranged from general to specific
- [ ] Use headings and subheadings to guide reader
- [ ] Headings are clear, descriptive, and parallel
- [ ] Include executive summary if longer than one page
- [ ] Provide table of contents if necessary

### Clarity & Precision
- [ ] Use short, active sentences
- [ ] Write in active voice
- [ ] Subject close to verb
- [ ] Avoid jargon or define terms when used
- [ ] Use plain, direct language
- [ ] Spell out abbreviations on first use
- [ ] Avoid ambiguity and misinterpretation
- [ ] Ensure logical flow and transitions

### Conciseness & Brevity
- [ ] Keep memo to one page if possible
- [ ] Be as short as possible while complete
- [ ] Remove unnecessary qualifiers and modifiers
- [ ] Eliminate wordiness and repetition
- [ ] Delete prepositional phrases in favor of adjectives/adverbs
- [ ] Remove weak verbs and nominalizations
- [ ] Use telegraphic style where appropriate
- [ ] Maximum 8 lines per paragraph (business standard)

### Tone & Style
- [ ] Maintain professional, cordial tone
- [ ] Use positive or neutral tone, avoid negative language
- [ ] Remain objective and unbiased
- [ ] Avoid personal opinions
- [ ] No slang or emojis
- [ ] More informal than external correspondence but still professional
- [ ] Courteous closing

### Formatting Consistency
- [ ] Consistent font throughout
- [ ] Consistent use of bold, italics, and capitalization
- [ ] Uniform bullet points and numbered lists
- [ ] Consistent date format
- [ ] Standard margins maintained
- [ ] Proper spacing between elements
- [ ] Professional font in black color

### Visual Design
- [ ] Use lists instead of paragraphs when possible
- [ ] Incorporate white space generously
- [ ] Use visual hierarchy (headings, bold, capitalization)
- [ ] Include charts, graphs, or tables for complex data
- [ ] Ensure graphics are high quality and labeled
- [ ] Make document skimmable

### Proofreading & Quality Control
- [ ] Proofread at least twice
- [ ] Take breaks between proofreading sessions
- [ ] Read document out loud
- [ ] Have another person review
- [ ] Check spelling, punctuation, and grammar
- [ ] Verify accuracy of numbers, dates, and data
- [ ] Ensure all URLs and contact information are correct
- [ ] Check for fragments, comma splices, run-ons
- [ ] Verify brand names and proper nouns are correct

### Closing & Action Items
- [ ] State what action reader should take
- [ ] Include sender's contact information
- [ ] Note any attachments or enclosures
- [ ] Provide clear call to action
- [ ] Include timeline for implementation if applicable

### Audience Considerations
- [ ] Tailor to audience's knowledge level
- [ ] Address three levels of readers (summary-only, skimmers, detailed readers)
- [ ] Consider reader's needs and background
- [ ] Write for decision-makers who are busy
- [ ] Include benefits to reader from desired actions

### Multi-Page Memos
- [ ] Include header on subsequent pages with recipient name, page number, and date
- [ ] Carry over at least two lines to second page
- [ ] Use plain paper for continuation pages

### Special Considerations
- [ ] Label CONFIDENTIAL in all caps if sensitive
- [ ] Use identification line (writer/typist initials) if applicable
- [ ] Include c: or cc: for copies, bc: or bcc: for blind copies
- [ ] For email memos, include greeting and omit full heading
